## What was built

T08 connected the LangGraph agent to Telegram: three files in `orchestrator/bot/`
handle all inbound messages and all outbound sends. `sender.py` is the single
exit point for Telegram API calls. `handlers.py` routes user text messages through
the agent; `buttons.py` handles Yes/No inline keyboard callbacks generated by
the EOD habit check-in.

T09 added three scheduled HTTP trigger endpoints in `orchestrator/proactive/triggers.py`:
morning task summary, end-of-day habit check-in (with inline Yes/No buttons), and
per-minute reminder firing. All three are called by cron-job.org and validated by
`PROACTIVE_SECRET` before taking any action.

## Why it was built

The system needs two interaction modes: reactive (user sends a message, agent replies)
and proactive (system initiates contact on a schedule). Without T08 the agent had no
Telegram interface. Without T09 there is no morning task summary, no EOD habit
check-in, and no mechanism to fire reminders at their scheduled time.

## Modules introduced or changed

| Module | Status | Description |
|---|---|---|
| `orchestrator/bot/sender.py` | new | Sole Telegram output channel; holds `_bot` reference set via `set_bot()` in lifespan |
| `orchestrator/bot/handlers.py` | new | Receives incoming text messages, passes to agent via `ainvoke`, replies via sender |
| `orchestrator/bot/buttons.py` | new | Handles `habit_log:<id>:yes/no` callback queries; dispatches yes-taps through agent |
| `orchestrator/proactive/triggers.py` | new | Three POST endpoints: `/trigger/morning`, `/trigger/eod`, `/trigger/check-reminders` |
| `orchestrator/main.py` | modified | Wires PTB Application into FastAPI lifespan; injects bot and agent graph at startup |
